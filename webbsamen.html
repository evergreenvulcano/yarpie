Webbsamen / repo-roll

Webbsamen bygger handtag där Storbastutinget lämnar byggbar takt.

Han bygger inte hela Yarpie och öppnar inte kärnan. Han gör HTML av slussade tingkonsekvenser: dörrar, fönster, statusar, 404:or, protokollkanter och återkommande format.

Arbetskedja

Från tingets kronologi till webbens handtag.

01

Storbastuting

Hjärtat samlar öden, företagsamhet, beslut, stopp och gemensam anda i tid.

02

Krönika

Sameräknaren säger vilka. Krönikan säger när, vad som stod på spel och vilken anda som bars vidare.

03

Sluss

Contentlogiken prövar mervärdet: byggbart, väntande, hållet internt eller spärrat.

04

Handtag

Webbsamen gör konsekvensen gripbar som sida, status, fönster, stängd dörr eller 404.

Byggbara format

Första paketet är nu ett ting-till-webb-paket.

Redaktionspaketet är sekundärt. Varje yta måste kunna peka bakåt mot ett ting, beslut, stopp, yrkande, andeförskjutning eller slussat mervärde innan den får webbform.

RUN C / lätt human review

Körd passagemodul

Fem utfall, fem ting-förankrade webbkonsekvenser.

Run Utfall Webbkonsekvens Läge
A press_release/regional_initiatives Händelsebar offentlighet efter ting, ingen extern release. kandidat, ej publicerad
B blocked_protected_source Avståendets etik: stoppet syns, materialet gör det inte. spärrad
C veckans_same Talskuldens återkommande format efter human review. redo för lätt review
H kontent_kronikor Yarpie-fönster: bredd mot stängda djup, utan kärna och utan analytics. installerad som fönsterlogik
J hold_internal Intern riktning blir stängd kant, inte innehåll. ingen produkt

Webbdisciplin

Det som inte är content kan ändå synas som kant.

RUN B och RUN J är tingets negativa makt. Systemet kan säga nej, hålla internt och låta en stängd passage vara den publika formen.
